Introduction

APIBridge API

APIBridge provides an OpenAI-compatible HTTP API to access Claude, GPT, Gemini, Mistral, and self-hosted models through a single endpoint. This page is the developer reference for integrating with our API.

Don't have a key yet? Sign up free — you'll get 10,000 tokens/day across all models.

Quickstart

Send your first request in three lines:

$ export APIBRIDGE_KEY=ak_live_xxxxxxxxxxxx
$ curl https://api.apibridge.io/v1/chat/completions \
    -H "Authorization: Bearer $APIBRIDGE_KEY" \
    -H "Content-Type: application/json" \
    -d '{"model":"claude-opus-4-7","messages":[{"role":"user","content":"Hello"}]}'

Sample response:

{
  "id": "chatcmpl-AbCdEfGhIj1234567890",
  "object": "chat.completion",
  "created": 1779000000,
  "model": "claude-opus-4-7",
  "choices": [{
    "index": 0,
    "message": { "role": "assistant", "content": "Hello! How can I help you today?" },
    "finish_reason": "stop"
  }],
  "usage": { "prompt_tokens": 8, "completion_tokens": 9, "total_tokens": 17 }
}

Authentication

All requests must include an Authorization header with your API key: 

Authorization: Bearer ak_live_xxxxxxxxxxxxxxxxxxxx

Key formats

PrefixMeaningNotes
ak_live_Production keyCounts against your billed quota.
ak_test_Sandbox keyFree, returns deterministic dummy responses for CI.
ak_user_Personal dev keyCapped at the free tier (10K tokens/day).
Treat your key like a password. Never commit it to source control. We rotate compromised keys automatically when detected on public GitHub.

Client libraries

APIBridge is a drop-in replacement for OpenAI. Existing OpenAI SDKs work without modification — just point them at our base URL.

Python
from openai import OpenAI
import os

client = OpenAI(
    base_url="https://api.apibridge.io/v1",
    api_key=os.environ["APIBRIDGE_KEY"],
)

resp = client.chat.completions.create(
    model="claude-opus-4-7",
    messages=[{"role": "user", "content": "Hello"}],
)
print(resp.choices[0].message.content)
Node.js
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://api.apibridge.io/v1",
  apiKey: process.env.APIBRIDGE_KEY,
});

const resp = await client.chat.completions.create({
  model: "claude-opus-4-7",
  messages: [{ role: "user", content: "Hello" }],
});
console.log(resp.choices[0].message.content);
Go
import openai "github.com/sashabaranov/go-openai"

cfg := openai.DefaultConfig(os.Getenv("APIBRIDGE_KEY"))
cfg.BaseURL = "https://api.apibridge.io/v1"
client := openai.NewClientWithConfig(cfg)

resp, err := client.CreateChatCompletion(ctx, openai.ChatCompletionRequest{
    Model: "claude-opus-4-7",
    Messages: []openai.ChatCompletionMessage{{Role: "user", Content: "Hello"}},
})

POST /v1/chat/completions

Generate a model response for a conversation.

Request body

FieldTypeDescription
modelstringRequired. Model ID, e.g. claude-opus-4-7.
messagesarrayRequired. Conversation history, each item has role and content.
max_tokensintMax tokens to generate. Defaults to model's max.
temperaturefloatSampling temperature 0–2. Default 1.0.
top_pfloatNucleus sampling. Default 1.0.
streamboolIf true, returns SSE stream. See Streaming.
toolsarrayFunction definitions. See Function calling.
response_formatobject{"type":"json_object"} for JSON mode.

Response

Returns a chat.completion object identical in shape to OpenAI's response.

Streaming

Set stream: true to receive a Server-Sent Events stream of incremental tokens. Useful for chat UIs that want to render mid-generation.

stream = client.chat.completions.create(
    model="claude-opus-4-7",
    messages=[{"role": "user", "content": "Write a haiku"}],
    stream=True,
)
for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")

The stream terminates with a data: [DONE] sentinel and the connection closes.

POST /v1/embeddings

Generate vector embeddings for semantic search and RAG. Defaults to text-embedding-3-large via OpenAI; we also proxy Voyage and Cohere.

resp = client.embeddings.create(
    model="text-embedding-3-large",
    input=["hello world", "foo bar baz"],
)
for e in resp.data:
    print(len(e.embedding))  # 3072

Function calling

Pass a tools array to let the model decide whether to invoke a function. The model returns a tool_calls array; you execute the function and feed the result back in the next turn.

tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "Get the current weather for a location",
        "parameters": {
            "type": "object",
            "properties": {"location": {"type": "string"}},
            "required": ["location"],
        },
    },
}]

resp = client.chat.completions.create(
    model="claude-opus-4-7",
    messages=[{"role": "user", "content": "What's the weather in Singapore?"}],
    tools=tools,
)

Rate limits

Limits are enforced per API key, per model, on a rolling 60-second window:

PlanRPMTPMConcurrent
Free2050,0005
Pro5002,000,00050
TeamCustomCustomCustom

Each response includes rate-limit headers:

x-ratelimit-limit-requests: 500
x-ratelimit-remaining-requests: 487
x-ratelimit-reset-requests: 12s
x-ratelimit-limit-tokens: 2000000
x-ratelimit-remaining-tokens: 1842919

Exceeding the limit returns HTTP 429 with Retry-After.

Errors

Errors follow OpenAI's shape:

{
  "error": {
    "message": "Invalid API key provided",
    "type": "invalid_request_error",
    "code": "invalid_api_key"
  }
}

400invalid_request_errorMalformed body, missing field, etc.
401invalid_api_keyKey missing, malformed, or revoked.
402quota_exceededFree tier daily cap hit. Upgrade or wait.
404model_not_foundUnknown model ID. Check /v1/models.
429rate_limit_exceededSlow down — see Retry-After.
500upstream_errorProvider returned an error.
503service_unavailableBrief incident — see status page.

Webhooks

Configure a webhook URL in the dashboard to be notified of:

  • quota.threshold — your spend reached 50/80/95% of plan
  • key.compromised — we detected your key on a public source
  • incident.opened / incident.resolved

Payloads are signed with HMAC-SHA256; verify the X-APIBridge-Signature header against your webhook secret.

Data policy

  • Requests are not used for training.
  • Logs retain the first 100 chars of prompt + completion for 30 days for debugging.
  • Set X-APIBridge-No-Log: true to opt out (Pro plan and up).
  • EU/SG data residency available on Team plan.
  • SOC2 Type II report available under NDA — email security@apibridge.io.
